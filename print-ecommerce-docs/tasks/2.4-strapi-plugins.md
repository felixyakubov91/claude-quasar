# Task 2.4: Strapi Plugins

## Goal
Configure Strapi plugins for i18n, SEO, and media uploads.

## Instructions

### Step 1: Install Plugins

```bash
cd backend
npm install @strapi/plugin-i18n @strapi/plugin-seo
npm install @strapi/provider-upload-cloudinary
```

### Step 2: Configure Plugins

Update `backend/config/plugins.js`:

```javascript
module.exports = ({ env }) => ({
  // i18n Plugin
  i18n: {
    enabled: true,
    config: {
      defaultLocale: 'he',
      locales: ['he', 'en'],
    },
  },

  // SEO Plugin
  seo: {
    enabled: true,
  },

  // Cloudinary Upload
  upload: {
    config: {
      provider: 'cloudinary',
      providerOptions: {
        cloud_name: env('CLOUDINARY_NAME'),
        api_key: env('CLOUDINARY_KEY'),
        api_secret: env('CLOUDINARY_SECRET'),
      },
      actionOptions: {
        upload: {},
        uploadStream: {},
        delete: {},
      },
    },
  },
});
```

### Step 3: Enable i18n on Content Types

In Strapi Admin → Settings → Internationalization:
1. Enable i18n for Product, Category, Homepage content types
2. Add Hebrew (he) and English (en) locales

## Acceptance Criteria
- [ ] i18n locales visible in content editor
- [ ] SEO fields available on content types
- [ ] Media uploads go to Cloudinary
